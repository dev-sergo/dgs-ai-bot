# Kickoff — следующая сессия разработки Dooglys AI-bot

> Вставить как первое сообщение новой сессии. Контекст и приоритеты зафиксированы
> по итогам сессии 2026-06-24 (боевая верификация + приёмочный прогон 100 кейсов).

---

Продолжаем разработку Dooglys AI-bot (Go-бэкенд, аналитика над POS-API Dooglys, локальные LLM,
sandwich-архитектура: числа считает движок, LLM — план + нарратив).

**Где мы:** ядро крепкое и проверено вживую на стенде (bot.bubnov.site). Подтверждено: выручка,
каналы оплаты, сравнение период-к-периоду (числа вменяемые), товары и drill-down, память диалога,
безопасность/отказы, живой Dooglys API (payment+products), Excel-выгрузка из UI (кнопка). Полный
журнал — `docs/05-quality-and-security-roadmap.md` (запись «2026-06-24 вечер»).

**Фокус этой сессии (по решению с пользователем):** движок + API на текущих данных, **анализатор**
и **консультант/рекомендации**. Slack и «ссылка на отчёт» — НЕ в работе (опровергнуты, канал — веб-чат).

**Приоритетные задачи** (детали — блок «Следующая сессия — приоритеты» N1–N6 в roadmap):
1. **N1 — консультант/подсказки:** свободные формулировки совета («где у меня проблемы», «как поднять
   средний чек», «что оптимизировать») сейчас уходят в off_topic-отписку вместо разбора. Сделать
   роутинг intent=advice устойчивым к парафразам.
2. **N2 — анализатор, направление:** «почему упала / что просело» отвечает «прирост», когда данные
   выросли — посылка пользователя не захватывается. Протащить «ожидаемое направление» plan→engine→нарратор.
3. **N3 — анализатор, доля:** «доля безналичных/онлайн» не считает процент. Дать derived-метрику доли канала.
4. **N4 — старт/ops:** `ProductIndex` в `cmd/server/main.go` блокирует старт HTTP на ~4 мин (синхронно,
   без таймаута). Строить индекс товаров в фоне/с таймаутом — сервер должен слушать сразу.
5. **N5 — даты:** месяц по названию («июнь», «с 1 по 15 июня») резолвится в 2023 год. Чинить к текущему году.
6. **N6 — опц.:** формат валюты (₽) на денежных колонках xlsx.

Начни с того, что предложишь порядок (что брать первым) и оценишь каждую задачу. Не приступай к коду,
пока не согласуем приоритет внутри списка.

## Рабочие правила (важно соблюдать)

- **Риг-прогоны запускаю Я (пользователь), не ты.** Ты готовишь Тир 0 (юнит/`make test`/`vet`) и отдаёшь
  команду; реальный LLM (`make eval-host`/`pipeval-host`) гоняю я и присылаю вывод.
- **Любая правка промпта планировщика → полный `eval-host` до коммита** (узкий фикс уже ронял 523→507).
  Чисто движковые правки — только `make test`. Go запускается через Docker (`make test`), бинаря `go` нет.
- **Прогоны сохранять в файл:** `… 2>&1 | tee <name>-$(date +%Y%m%d-%H%M).log`.
- **Git-коммиты на английском, без Co-Authored-By.** Коммитить/пушить — только когда прошу.
- **Стенд:** деплой `docker compose up --build` запускаю я; бэкенд `bot` на :8088, веб на :8090,
  снаружи — bot.bubnov.site (туннель). Токен демо-гейта — у меня.
- **Ручной тест стенда** курлами по `/api/ask` и `/api/export` ты можешь гонять сам (браузерный
  User-Agent, токен в `X-Auth-Token`); harness probe-скрипты — в `test/manual/`.

## Карта проекта (куда смотреть)
- `internal/engine/` — детерминированные расчёты (Plain/TopN/Compare/Contribution); `classb.go` — анализ.
- `internal/planner/llm.go` + `stub.go` — план из запроса (промпт + роутинг); `internal/plan` — схема/валидатор.
- `internal/advisor/` — консультант (LLM поверх InsightBundle); `internal/narrator/` — нарратив.
- `internal/dooglys/` — источник данных (API/фикстуры); `cmd/server/main.go` — старт (см. N4).
- `web/` — фронт (React); `docs/` — видение, архитектура, roadmap, дизайн консультанта.

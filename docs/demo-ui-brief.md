---

Ты — фронтенд-инженер. Сделай **одностраничное демо-приложение** для AI-ассистента по аналитике
кафе (платформа Dooglys). Цель — красиво показать первую итерацию: пользователь вводит запрос
обычным текстом, видит загрузку, затем плавно «печатается» ответ и аккуратно отрисовывается
таблица с цифрами.

## Что уже есть (бэкенд — НЕ трогать, он готов)

Go-сервис отдаёт один эндпоинт `POST http://localhost:8088/ask`.

**Запрос:**
```http
POST /ask
Content-Type: application/json
X-Tenant-ID: mock_single        // опционально; по умолчанию mock_single

{ "text": "почему упала выручка за месяц" }
```

**Ответ (200)** — JSON. Главные поля для UI: `answer` (готовый текст), `envelope`
(структурированный результат для красивой таблицы), `validation` (ветка уточнения).

### Ошибка
HTTP 4xx/5xx, тело `{ "error": "текст" }`.

## Логика отрисовки ответа

1. `validation.NeedClarify === true` → показать `ClarifyPrompt` как вопрос-подсказку (мягкий стиль), без таблицы.
2. `envelope` присутствует → главный путь:
   - если `envelope.narrative` не пустой (Class B) — **его печатать typewriter-эффектом** как заголовок-объяснение;
     иначе сгенерировать короткий заголовок из `summary` (например: «Выручка за неделю: 1 041,74 ₽»);
   - под заголовком — **карточки-итоги** из `summary` (chips);
   - затем **таблица** из `columns` + `rows` с плавным появлением строк.
3. Тело `error` → аккуратный тост/баннер с текстом.

## Форматирование чисел (важно — значения в JSON «сырые»)

Числа в `rows`/`summary` — обычные числа (`510.74`). Форматируй по `unit` колонки (RU-стиль:
пробел-разделитель тысяч, запятая-десятичные):
- `RUB`   → `1 041,74 ₽`  (2 знака)
- `count` → `6`           (целое)
- `percent` → `25,20 %`   (2 знака)
- `date`  → как есть (`2026-06-18`) или дружелюбно `18.06.2026`
- `string`→ как есть
Отрицательные — со знаком (`-832,65 ₽`). Можно подсвечивать рост/падение цветом (зелёный/красный)
по знаку `delta`/`delta_abs`.

## UX

- **Один крупный центрированный инпут** (как поиск), плейсхолдер вроде «Спросите про выручку, товары, динамику…».
- Под инпутом — 3–4 **кликабельных примера-чипа**: «выручка за неделю», «почему упала выручка за месяц»,
  «топ товаров за неделю», «сравни выручку этой недели».
- Submit по Enter и по кнопке.
- **Состояние загрузки**: бэкенд отвечает 3–5 секунд — покажи приятный лоадер (шиммер/пульс + «Анализирую…»),
  не блокируй ввод грубо.
- **Появление ответа**: typewriter для заголовка/нарратива, затем fade-in таблицы и chips.
- Сохрани историю запросов в сессии (лента карточек «запрос → ответ»), чтобы можно было листать на демо.
- Адаптивность: ок на ноутбуке; мобайл — бонус.

## Стиль — под Dooglys

Открой https://dooglys.com и подхвати визуальный язык: чистый современный SaaS, светлая тема,
скруглённые карточки, мягкие тени, акцентный фирменный цвет, много воздуха, аккуратная типографика
(моноширинный/таблично-выровненный шрифт для цифр — приятно для аналитики). Сделай со вкусом,
не перегружай. Логотип/название — «Dooglys AI» в шапке.

## Технологии и запуск

- **Vite + React + TypeScript** (или Vue — на твой выбор). Без бэкенда внутри UI.
- **CORS:** наш Go-сервис может не отдавать CORS-заголовки. Чтобы не упереться в это — настрой
  **dev-прокси Vite**: фронт зовёт относительный `/api/ask`, Vite проксирует на `http://localhost:8088`.
  Пример `vite.config.ts`:
  ```ts
  server: { proxy: { '/api': { target: 'http://localhost:8088', changeOrigin: true,
    rewrite: p => p.replace(/^\/api/, '') } } }
  ```
  Тогда в коде делай `fetch('/api/ask', { method:'POST', headers:{'Content-Type':'application/json','X-Tenant-ID':'mock_single'}, body: JSON.stringify({text}) })`.
- Поддержи переменную `VITE_API_BASE` (по умолчанию `/api`) — на случай прямого адреса.
- Deliverables: рабочий `npm install && npm run dev`, короткий README с командами.

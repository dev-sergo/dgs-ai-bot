# 05 — Дорожная карта качества и безопасности

Живой документ. Ведём по нему доработки **от простого к сложному** и фиксируем **каждую итерацию**,
чтобы всегда было видно: где мы сейчас, что делали в прошлый раз, что дальше.

## Как пользоваться

1. Берём следующую невыполненную задачу из чек-листа (сверху вниз — порядок = приоритет).
2. Делаем, прогоняем тесты/бенчмарк.
3. **Дописываем строку в «Журнал итераций»** (сверху, новые — выше): дата, что сделали, результат.
4. Отмечаем задачу `[x]` и ставим дату/коммит.

Метрика прогресса — pass-rate бенчмарка `make eval-host` (см. [03-testing-strategy.md](03-testing-strategy.md)).
Текущий baseline зафиксирован ниже.

## Принципы (не нарушать при доработках)

- LLM не считает числа — вся математика детерминирована в Go.
- `tenant_id` и токены — только server-side, в план модели не попадают.
- Любой выход за white-list → отказ/уточнение до обращения к данным.
- Лучше честное «не умею / уточните», чем уверенно неверный ответ.
- Каждая доработка сопровождается кейсами в бенчмарке и/или тестами.

# Dooglys AI-bot — что умеет и куда развивается

---

## Коротко

AI-ассистент по аналитике заведения: вы спрашиваете **обычными словами** —
«сколько выручки за неделю», «какой товар просел», «на чём я теряю» — и получаете
**точный ответ по реальным данным Dooglys**, с возможностью **выгрузить в Excel**.

Ключевое отличие: **цифры считает не нейросеть, а детерминированный движок**.
Нейросеть только понимает вопрос и формулирует ответ словами — поэтому «придумать»
или исказить число она не может. Это разбор бизнеса, которому можно доверять.

## Как это выглядит в диалоге

> **— Сколько выручки за прошлую неделю?**
>
> За 16–22 июня — **412 800 ₽** по завершённым заказам (1 240 чеков, средний чек 333 ₽).
> Каналы оплаты: карта 58%, наличные 27%, онлайн 11%, СБП 4%. Возвраты — 6 200 ₽.
>
> **— Выросла или упала к позапрошлой неделе?**
>
> Выручка выше на **6%** (+23 100 ₽), средний чек почти не изменился.
>
> **— Сколько за этот период продали Бизнес-ланча?**
>
> Уточните, какой именно: «Все включено», «Лёгкий» или «Оптимальный»?
>
> **— Оптимальный.**
>
> Бизнес-ланч «Оптимальный»: **318 порций** на **47 700 ₽**, со скидками на 2 100 ₽.
> *Любой такой отчёт выгружается в Excel одной кнопкой.*
>
> *(числа иллюстративные; на стенде ответы строятся на живых данных Dooglys
> и совпадают с независимой проверкой — см. «Что доказывает пилот»)*

## Что умеет сейчас

### Аналитика — 4 отчёта
| Отчёт | Что показывает | Источник |
|-------|----------------|----------|
| **Выручка** | по дням: чеки, нал / карта / онлайн / СБП, возвраты, средний чек | 🟢 живой API |
| **Товары** | продажи, количество, выручка, скидки; разбор по конкретному товару | 🟢 живой API |
| **Чеки** | отдельные транзакции | тестовый набор |
| **Заказы** | доставка / самовывоз, источники, статусы | тестовый набор |

### Способы анализа — к любому отчёту
- **Сводка за период** с итогами (сегодня, неделя, месяц, произвольные даты)
- **Рейтинги** — топ и худшие (товары, точки)
- **Сравнение период-к-периоду** — «неделя к неделе», «месяц к месяцу»: динамика
  выручки и среднего чека (рост/падение, абсолют и %).
- **Разбор причин** — вклад каналов оплаты (карта / наличные / онлайн / СБП) в изменение
  выручки. Причинные формулировки и разрез «кто» (по сотрудникам) — в плане, см. ниже.

### Консультант
- На вопросы вида «**на чём я теряю**», «**что улучшить**» ассистент уточняет период
  и собирает связный разбор поверх чисел: возвраты, скидки, средний чек, структура
  каналов оплаты, товары-аутсайдеры.
- Понимание более свободных формулировок расширяется итеративно (см. план).

### Живой диалог
- Вопросы-вдогонку: «а за прошлый месяц?», «а по точке Выкса?» — помнит контекст.
- Переспрашивает, когда запрос неоднозначен, — не угадывает вслепую.
- Возобновляет разбор после уточнения периода.

### Данные и выгрузка
- Работает на **живом JSON API Dooglys** (официальный интерфейс, не парсинг страниц).
- **Экспорт отчёта в Excel (.xlsx)** одним скачиванием.

## Как это устроено (коротко)

Четыре слоя, каждый со своей зоной ответственности:

```
Вопрос → [1] Планировщик (LLM)  — понимает запрос, строит план
       → [2] Движок (детермин.) — считает числа по данным Dooglys
       → [3] Нарратор (LLM)     — описывает результат словами
       → Ответ + Excel
         [4] Слой данных        — живой API Dooglys (token-auth)
```

**Принципы безопасности, заложенные в фундамент:**
- **Sandwich**: LLM окружает математику, но не делает её. Числа — только из движка,
  «галлюцинация цифры» исключена архитектурно.
- **Защита персональных данных**: кассиры, телефоны, ФИО клиентов не попадают в ответы.
- **Изоляция заведений**: архитектура разделяет данные по тенантам (сейчас
  демонстрируется на одном); доступ к чужим данным закрыт by design.
- **Вход под токеном**: сервис закрыт от посторонних.
- **Локальные LLM**: данные не уходят во внешние облака.

## Что доказывает пилот

На боевых данных тестового тенанта система сквозным путём (живой API → расчёт → ответ)
строит отчёты **«Выручка»** и **«Товары»**: выручка — с разбивкой по дням, каналам оплаты
и возвратам (итоги **совпадают с независимой проверкой до копейки**); товары — с разбором
по конкретной позиции («сколько продали Бизнес-ланча за май»). Правило выручки согласовано с
заказчиком (с учётом доставки, по завершённым заказам, возвраты вычитаются).

## Где мы сейчас — текущая итерация

Закрыто в этой итерации:
- **«Товары» переведены на живой API** (из тех же заказов, что и выручка) + разбор по
  конкретному товару.
- **Сравнение и средний чек откалиброваны на боевых данных**: динамика выручки и среднего
  чека между периодами считается верно; корректно распознаётся «прошлая неделя» (отдельно
  от «за неделю»); вопрос «сколько чеков» отвечает количеством.
- **Консультант откалиброван на боевых данных**: спрашивает период, если его не назвали;
  доля возвратов и каналов считается корректно; не выдумывает «убыточные» позиции там,
  где нет данных о себестоимости.
- **Демо-доступ под токеном** и **экспорт отчётов в Excel** (выручка, товары).
- Сборка разворачивается на стенде по ссылке.

## Следующие итерации (план)

1. **Демо-обёртка и сценарий показа** — стенд по ссылке + короткий сквозной сценарий
   для заказчика (вопрос → отчёт → разбор → Excel).
2. **«Чеки» и «Заказы» на живой API** — последние два отчёта с тестовых наборов на боевые,
   без изменения остальной системы.
3. **Углублённый разбор причин и аналитика персонала** — причинные формулировки
   («почему именно упала выручка» с верным направлением и вкладом), а также выручка и
   средний чек по кассирам/сменам, доля возвратов и скидок в разрезе сотрудников
   (нужен разрез по персоналу в данных).
4. **Шире понимание формулировок** — консультант и отчёты уверенно отвечают на больший
   круг свободных вопросов («где у меня проблемы», «что оптимизировать», запросы с
   произвольными датами), без привязки к точным фразам.
5. **Прогноз, тренды и авто-алерты** — прогноз выручки/спроса, выделение трендов и
   автоматические уведомления об аномалиях (всплеск возвратов, провал выручки на точке).

# Dooglys AI-bot — что умеет и куда развивается

> **Кому:** заказчику (ЛПР) и команде Dooglys.
> **Зачем:** одним документом показать, что система уже делает на боевых данных,
> как она устроена (коротко) и что предлагаем разработать дальше.

---

## Коротко

AI-ассистент по аналитике заведения: вы спрашиваете **обычными словами** —
«сколько выручки за неделю», «какой товар просел», «на чём я теряю» — и получаете
**точный ответ по реальным данным Dooglys**, с возможностью **выгрузить в Excel**.

Ключевое отличие: **цифры считает не нейросеть, а детерминированный движок**.
Нейросеть только понимает вопрос и формулирует ответ словами — поэтому «придумать»
или исказить число она не может. Это разбор бизнеса, которому можно доверять.

---

## Что умеет сейчас

### Аналитика — 4 отчёта
| Отчёт | Что показывает |
|-------|----------------|
| **Выручка** | по дням: чеки, нал / карта / онлайн / СБП, возвраты, средний чек |
| **Товары** | продажи, количество, скидки, ожидаемая прибыль |
| **Чеки** | отдельные транзакции |
| **Заказы** | доставка / самовывоз, источники, статусы |

### Способы анализа — к любому отчёту
- **Сводка за период** с итогами (сегодня, неделя, месяц, произвольные даты)
- **Рейтинги** — топ и худшие (товары, точки)
- **Сравнение период-к-периоду** — «неделя к неделе», «месяц к месяцу»
- **Разбор причин** — «что/кто виноват в падении выручки» (вклад компонент в изменение)

### Консультант
- Вопрос «**на чём я теряю / что улучшить**» → связный разбор поверх чисел:
  возвраты, скидки, средний чек, структура каналов оплаты, товары-аутсайдеры.

### Живой диалог
- Вопросы-вдогонку: «а за прошлый месяц?», «а по точке Выкса?» — помнит контекст.
- Переспрашивает, когда запрос неоднозначен, — не угадывает вслепую.
- Возобновляет разбор после уточнения периода.

### Данные и выгрузка
- Работает на **живом JSON API Dooglys** (официальный интерфейс, не парсинг страниц).
- **Экспорт отчёта в Excel (.xlsx)** одним скачиванием.

## Как это устроено (коротко)

Четыре слоя, каждый со своей зоной ответственности:

```
Вопрос → [1] Планировщик (LLM)  — понимает запрос, строит план
       → [2] Движок (детермин.) — считает числа по данным Dooglys
       → [3] Нарратор (LLM)     — описывает результат словами
       → Ответ + Excel
         [4] Слой данных        — живой API Dooglys (token-auth)
```

**Принципы безопасности, заложенные в фундамент:**
- **Sandwich**: LLM окружает математику, но не делает её. Числа — только из движка,
  «галлюцинация цифры» исключена архитектурно.
- **Защита персональных данных**: кассиры, телефоны, ФИО клиентов не попадают в ответы.
- **Изоляция заведений**: данные одного тенанта недоступны другому.
- **Вход под токеном**: сервис закрыт от посторонних.
- **Локальные LLM**: данные не уходят во внешние облака.

## Что доказывает пилот

На боевых данных тестового тенанта система сквозным путём (живой API → расчёт → ответ)
строит отчёт «Выручка» с разбивкой по дням, каналам оплаты и возвратам, а итоги
**совпадают с независимой проверкой до копейки**. Правило выручки согласовано с
заказчиком (с учётом доставки, по завершённым заказам, возвраты вычитаются).

## Дорожная карта (предложения по развитию)

### A. Боевые отчёты «Товары» и «Персонал» на API
Сейчас на живых данных работает «Выручка»; «Товары» и «Персонал» — на тестовых
наборах. Подключаем их к тому же API → полная аналитика ассортимента и сотрудников
на реальных данных, без изменения остальной системы.

### B. Прогноз, тренды и авто-алерты
- Прогноз выручки и спроса на ближайший период.
- Выделение трендов («третью неделю растёт средний чек»).
- **Автоматические уведомления об аномалиях**: резкий рост возвратов, провал
  выручки на точке, всплеск скидок — система сама пишет, не дожидаясь вопроса.

### C. Аналитика персонала и смен
- Выручка и средний чек **по кассирам и сменам**.
- КПД сотрудников, доля возвратов/скидок в разрезе персонала.
- Основа для мотивации и контроля — кто приносит результат, где теряем на сменах.

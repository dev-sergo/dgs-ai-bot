# `bench/` — история замеров качества

Здесь хранятся **прогоны бенчмарков** — главный артефакт прогресса/регресса продукта.
Это не временные логи: каждый прогон фиксирует, как система отвечала на корпус вопросов
на конкретном коммите. По ним видно, движемся мы вперёд или назад.

> Логи процесса (старт сервера/бота, `go test`/`vet`, API-смоук) сюда **не** кладутся —
> они эфемерны и игнорируются (`.gitignore`). Сюда попадает только то, что даёт **оценку против корпуса**.

## Что где

| | |
|---|---|
| [`LEDGER.md`](LEDGER.md) | **главное**: таблица тренда, одна строка на прогон |
| `runs/` | сами прогоны: пара `.log` (сырьё) + `.json` (сводка) на каждый |
| `summarize.sh` | хелпер: из `.log` → `.json`-сводка |

## Контракт имени

```
runs/<YYYY-MM-DD>_<HHMM>_<harness>_<corpus>_<mode>_<commit>.{log,json}
        │            │       │         │        │       └ короткий git SHA — привязка к коду
        │            │       │         │        └ llm | stub
        │            │       │         └ prompts | pipeline | followups | personnel | filter-values | quality
        │            │       └ eval | pipeval
        │            └ время старта (HHMM)
        └ дата (сортировка по имени = хронология)
```

- **harness** — `eval` (проверяет план: intent/report/method/period/filters) или `pipeval` (проверяет финальный ответ: текст/числа/envelope).
- **corpus** — какой набор кейсов гонялся (файлы в [`test/eval/`](../test/eval/)).
- **mode** — `llm` (реальная модель) или `stub` (детерминированный, для CI).
- **commit** — состояние кода. Даёт воспроизводимость: можно вернуться на `git checkout <sha>`.

## Как добавить новый прогон

1. Прогнать с сохранением в файл сразу по контракту имени:
   ```bash
   make eval-host 2>&1 | tee bench/runs/$(date +%Y-%m-%d_%H%M)_eval_prompts_llm_$(git rev-parse --short HEAD).log
   ```
2. Сгенерировать сводку и дописать строку в леджер:
   ```bash
   f=bench/runs/<имя>.log
   bench/summarize.sh "$f" > "${f%.log}.json"
   ```
   Затем добавить строку в [`LEDGER.md`](LEDGER.md) (значения берутся из `.json`).

## Метрики

- **eval**: `прошло` (pass) — кейсы, где план совпал полностью; `валидных` (valid) — где план структурно корректен. Заголовок тренда = `прошло/total`.
- **pipeval**: `прошло/total` — кейсы, где финальный ответ прошёл все assert'ы.
- Латентность `p50/p95` в мс (для `mode=stub` нули — модель не вызывается).
